# AI Gateway 前端 PRD

## 1. 产品概述

AI Gateway 前端包含两个独立的 React 应用：**用户端**（聊天界面）和**管理端**（运维监控面板）。用户端面向普通用户，提供与 AI 模型的对话交互；管理端面向运维人员，提供模型健康监控、槽位状态管理和日志查看。

- 用户端：让用户通过 OpenAI 兼容接口与轻量/大模型对话，支持流式输出
- 管理端：实时监控模型后端健康状态、排队情况，手动触发健康检查

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 普通用户 | 使用聊天界面 | 发起对话、查看历史 |
| 管理员 | 使用管理后台 | 查看健康状态、触发检查、查看槽位 |

### 2.2 功能模块

#### 用户端（端口 3000）
1. **聊天页**：对话输入框、流式消息展示、模型路由指示（显示当前使用轻量/大模型）

#### 管理端（端口 3001）
1. **仪表盘页**：模型健康状态卡片、槽位使用情况、最近请求统计
2. **模型管理页**：模型列表、手动触发健康检查、查看检查历史

### 2.3 页面详情

#### 用户端 - 聊天页

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 聊天页 | 消息列表 | 展示用户和 AI 的对话记录，支持 Markdown 渲染 |
| 聊天页 | 输入区域 | 文本输入框、发送按钮、流式输出时显示停止按钮 |
| 聊天页 | 模型指示器 | 显示当前请求被路由到轻量模型还是大模型 |
| 聊天页 | 对话管理 | 新建对话、清空对话历史 |

#### 管理端 - 仪表盘页

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 仪表盘 | 健康概览 | 三个模型后端（轻量/大模型/评估模型）的健康状态卡片 |
| 仪表盘 | 槽位监控 | 大模型当前槽位使用情况，活跃数/等待数/总数 |
| 仪表盘 | 状态时间线 | 最近健康检查结果的变化记录 |

#### 管理端 - 模型管理页

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 模型管理 | 模型列表 | 显示所有模型名称、地址、健康状态、响应耗时 |
| 模型管理 | 手动检查 | 一键触发所有模型健康检查 |
| 模型管理 | 模型详情 | 展示单个模型的错误信息、上次检查时间 |

## 3. 核心流程

### 用户端聊天流程

用户输入消息 → 前端发送 POST /v1/chat/completions (stream=true) → 接收 SSE 流 → 实时渲染 AI 回复 → 显示路由标签（light/big）

```mermaid
flowchart TD
    "A[用户输入消息]" --> "B[发送请求到 Gateway]"
    "B" --> "C{复杂度判断}"
    "C" --> "|简单| D[轻量模型处理]"
    "C" --> "|复杂| E[大模型处理（排队）]"
    "D" --> "F[SSE 流式返回]"
    "E" --> "F"
    "F" --> "G[前端实时渲染回复]"
```

### 管理端监控流程

管理员打开仪表盘 → 自动轮询 /health 获取状态 → 展示健康/异常 → 手动触发检查 → 更新状态

```mermaid
flowchart TD
    "A[管理员打开仪表盘]" --> "B[自动轮询 /health]"
    "B" --> "C[展示模型健康状态]"
    "C" --> "D{发现异常?}"
    "D" --> "|是| E[高亮异常模型]"
    "D" --> "|否| F[显示正常状态]"
    "E" --> "G[点击手动检查]"
    "G" --> "H[POST /health/check]"
    "H" --> "C"
```

## 4. 用户界面设计

### 4.1 设计风格

**用户端**：深色主题，科技感，简洁高效
- 主色：深灰 #0f0f0f 背景 + 翠绿 #10b981 强调色
- 圆角卡片，微妙的玻璃态效果
- 字体：JetBrains Mono（代码/消息）+ Noto Sans SC（UI 文字）

**管理端**：深色运维面板风格，数据密集
- 主色：深蓝灰 #1e1e2e 背景 + 琥珀 #f59e0b 警告色 + 绿 #22c55e 正常色
- 数据卡片布局，实时刷新动画
- 字体：JetBrains Mono + Noto Sans SC

### 4.2 页面设计概览

#### 用户端 - 聊天页

| 模块 | UI 元素 |
|------|---------|
| 消息列表 | 深色背景，用户消息右对齐绿色气泡，AI 消息左对齐灰色气泡，Markdown 渲染 |
| 输入区域 | 底部固定，圆角输入框 + 发送按钮，流式输出时显示加载动画 |
| 模型指示器 | 消息气泡上方小标签，显示 "轻量模型" 或 "大模型" |

#### 管理端 - 仪表盘页

| 模块 | UI 元素 |
|------|---------|
| 健康卡片 | 三列网格，每张卡片显示模型名、状态圆点（绿/红）、响应耗时 |
| 槽位监控 | 进度条 + 数字，显示 active/available/waiting |
| 手动检查按钮 | 琥珀色按钮，点击后刷新所有卡片 |

### 4.3 响应式

- 桌面优先，适配 1280px+ 宽度
- 移动端基础适配（单列布局）
